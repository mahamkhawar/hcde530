# Week 2 — Competency 2: Code literacy and documentation

Personal reflections from this week’s work (CSV + Python scripts + optional HTML).

## What I learned about code (reading / structure)

Coding languages have strict structures and syntax. That specificity is what tells the computer exactly what to perform. AI tools have been helpful for understanding those nuances—for example, why **indentation** matters in Python (it is part of the structure, not just formatting).

## What I learned about documentation

Comments matter—not only for other people, but for **myself**, so I can remember **how** and **why** something was done. Comments can get detailed about what a line does and how it fits into the overall program. Going forward, I want to keep the habit of commenting **why**, not just **what**.

## Something in my code that “clicked”

In `demo_word_count.py`, the parts that print the table output make more sense now. At first it was not intuitive, but I understand better how the script builds a readable summary (including how output is organized before and inside the loop that goes through each row).

## What felt hardest this week

The hardest part was not a single Python line—it was understanding how **Cursor**, the **terminal**, **Git**, and **GitHub** fit together on my computer versus “just” working inside the IDE. That bigger system picture is still fuzzy.

## Where I am with running Python

On a 1–5 scale of comfort running a Python script and reading the output, I am about a **2**. I am still rusty with Python in general (I last used it in undergrad about four years ago).
